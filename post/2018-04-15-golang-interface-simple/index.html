Golang interface入门

 2018.4.15  2021.2.20  Post  1397  

定义

Interface是定义一组方法的集合

如何实现接口

任何type只要实现了interface的所有方法, 即可实现接口. 不需要像java需要使用implements关键字来显式声明.

所有的类型都实现了empty interfa1ce

type I interface {
	M()
}

type T struct {
	S string
}

// T就实现了interface I, 不需要显式声明
func (t T) M() {
	fmt.Println(t.S)
}

func main() {
	var i I = T{"hello"}
	i.M()
}

type I interface {
	M()
}

type T struct {
	S string
}

func (t *T) M() {
	fmt.Println(t.S)
}

type F float64

func (f F) M() {
	fmt.Println(f)
}

func main() {
	var i I

	i = &T{"Hello"}
	describe(i)
	i.M()

	i = F(math.Pi)
	describe(i)
	i.M()
}

func describe(i I) {
	fmt.Printf("(%v, %T)\n", i, i)
}

interface值是nil

如果interface的值是nil, 那么实现这个接口的类型的receiver也是nil

在其他语言中, 用空指针或者空对象调用对象的函数, 这将会抛出异常. 但是在go中这是很普遍的, 可以被完美的解决, 例如像下面的M一样.

type I interface {
	M()
}

type T struct {
	S string
}

func (t *T) M() {
	if t == nil {
		fmt.Println("<nil>")
		return
	}
	fmt.Println(t.S)
}

func main() {
	var i I
	var t *T
	i = t
	describe(i)
	i.M()
}

func describe(i I) {
	fmt.Printf("(%v, %T)\n", i, i)
}

interface 类型断言

1

t := i.(T)

断言表达式的意思: iinterface的值的具体类型是T, 并且把类型为T的值赋值给变量t. 如果interface i不是T这个类型, 那么这个表达式会panic

1

t, ok := i.(T)

判断interface是否是指定的类型, 类型断言返回两个结果: 值 和 boolean.

如果i是T类型, 那么t是具体的值, ok是true

如果i不是T类型, 那么t将会是类型T的0值, ok是false, 不会有panic发生.

	var i interface{} = "hello"

	s := i.(string)
	fmt.Println(s)

	s, ok := i.(string)
	fmt.Println(s, ok)

	f, ok := i.(float64)
	fmt.Println(f, ok)

	f = i.(float64) // panic
	fmt.Println(f)
}

type Person struct {
	Name string
	Age  int
}

func (p Person) String() string {
	return fmt.Sprintf("%v (%v years)", p.Name, p.Age)
}

func main() {
	a := Person{"Arthur Dent", 42}
	z := Person{"Zaphod Beeblebrox", 9001}
	fmt.Println(a, z)
}

type ErrNegativeSqrt float64

func (e ErrNegativeSqrt) Error() string {
	return fmt.Sprintf("cannot Sqrt negative number: %f", e)
}

func Sqrt(x float64) (float64, error) {
	if x < 0 {
		return x, ErrNegativeSqrt(x)
	}
	return x, nil
}

func main() {
	fmt.Println(Sqrt(2))
	fmt.Println(Sqrt(-2))
}
